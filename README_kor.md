# Tractatus Dialogicus 기술 명세 (v1.8)

본 애플리케이션은 **Gemini 3 Pro**의 강력한 추론 능력을 활용하여, 인공지능의 내부 사고 과정(메타인지)을 투명하게 관측하고 이를 명제화하는 **분석적 메타인지 인터페이스**입니다.

## 1. 핵심 동작 메커니즘: 3단계 생성 프로세스 (A/B/C)

단순한 질의응답을 넘어, 본 시스템은 인공지능의 '사유'와 '발화'를 엄격히 분리하는 3단계 워크플로우를 강제합니다.

### **Phase A: Latent Metaphysics (사유의 합성)**
*   **목적**: 사용자의 입력을 논리적 객체로 분석하고, 답변을 내놓기 전 내부적인 추론 경로를 먼저 구축합니다.
*   **방법**: `gemini-3-pro-preview` 모델의 `thinkingBudget`(최대 32,768 토큰)을 풀가동하여, 논리의 한계와 의미 구조를 탐색합니다.
*   **결과**: 사용자에게는 보이지 않는(또는 Trace 패널에서만 보이는) 'Reasoning Trace'가 생성됩니다.

### **Phase B: Structural Manifestation (명제의 표명)**
*   **목적**: Phase A에서 도출된 사유 결과를 바탕으로, 사용자에게 전달될 최종 명제를 작성합니다.
*   **방법**: 이전 대화 이력과 Phase A의 추론 결과를 컨텍스트로 주입하여, 사유와 표명 사이의 논리적 정합성을 확보합니다.
*   **결과**: 화면에 번호(N.1)와 함께 표시되는 최종 답변입니다.

### **Phase C: Bilingual Parity Sync (한/영 논리적 동기화)**
*   **목적**: 한국어와 영어 환경 사이의 대화 이력이 단 토큰의 오차 없이 동일한 논리적 상태를 유지하도록 합니다.
*   **방법**: `gemini-3-flash-preview`를 사용하여 사용자의 질문, 모델의 답변, 그리고 내부 추론(Reasoning)까지 통째로 번역하여 반대편 언어 히스토리에 즉시 주입합니다.
*   **결과**: 언어를 전환해도 동일한 분석 맥락이 유지됩니다.

## 2. 비트겐슈타인식 계층적 명제 구조

본 인터페이스는 루드비히 비트겐슈타인의 *논리-철학 논고(Tractatus Logico-Philosophicus)*의 표기법을 오마주하여 대화를 정형화합니다.

- **1.1**: 시스템 초기 공리 (Greeting)
- **N**: 주관적 사용자 입력 (Subjective Input)
- **N.1**: 객관적 분석 명제 (Manifested Proposition)

이러한 번호 체계는 단순한 UI 요소가 아니라, 대화가 누적됨에 따라 세계의 논리적 지도가 확장되는 과정을 시각화합니다.

---

## 3. 기술적 특징 및 복원력

- **Logic Saturation (429 에러) 대응**: 고부하 추론 작업 시 발생하는 API 할당량 제한을 해결하기 위해 지터(Jitter)가 포함된 지수 백오프(Exponential Backoff) 재시도 로직을 구현했습니다.
- **메모리 무결성**: 모델의 과거 추론 내역(`Reasoning Trace`)을 다음 턴의 컨텍스트로 재주입함으로써, 인공지능이 자신의 이전 사유를 기억하고 그 위에서 논리를 쌓아올리는 '재귀적 인지'가 가능합니다.
- **순수 메타인지 관측**: 인위적인 캐릭터 연기나 환각을 최소화하도록 시스템 프롬프트를 설계하여, LLM의 실제 연산 과정을 공정하게 관측하는 데 집중합니다.

---

## 4. 데이터 관리

- **로컬 세션**: 최대 10개의 독립적인 논리 스트림을 브라우저 로컬 저장소에 보관합니다.
- **Truth Stream Export**: 전체 대화 내역(추론 데이터 포함)을 JSON 형식으로 내보내어 분석의 재현성과 투명성을 보장합니다.
